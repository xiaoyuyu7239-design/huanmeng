## 交付流程

1. 生图管线（妙搭 AI生图插件或人工外包）就绪后，按 P0→P1→P2 顺序产出；
2. 每张经人工目检（着装/构图/风格协调）后放入 公共资源/portraits/<slug>/；
3. 在对应 story.json 的 cast 中填 portrait 字段；
4. 跑 `npm run test:assets` 验证引用完整，`npm test` 全量回归。

## 生图规则增补（2026-07-17 产品反馈：多人场景人物模糊、五官缺失）

1. **禁止多人中远景**：全景/场景图中人物数量 ≤2，且人物占画面高度 ≥40%，否则 AI 生成的面部必然崩坏。
2. 群像需求一律拆解为：纯场景图 + 单人特写立绘分开生成，由界面层叠加呈现。
3. 现有多人图的过渡策略：初始视角避开人群（entryView 调整）、文案将无脸人形合理化（模特/剪影/背影/数据投影），已在《夜航电台》《第七织女》应用。
4. 待重制清单（含人群缺陷）：s07-broadcast-countdown（台上八人）、s11-rooftop-confrontation（天台三人）、project-20260620-002835-s00-arrival（十家传人群像）。
