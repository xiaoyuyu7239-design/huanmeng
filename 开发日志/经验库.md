# 经验库

> 每轮循环回答三问：哪里返工了？为什么？下轮怎么避免？沉淀为编号经验，开工前必读。

- **E-001**（第1轮）从 JS 里正则提取素材路径时，模板字符串 `${变量}` 会被误当成真实路径。
  **为什么**：正则只认文本形状，不懂 JS 语法。
  **下轮怎么避免**：提取结果先过滤含 `${` 的路径再下载；判断"下载失败是否要紧"时先看路径是不是真文件名。

- **E-002**（第1轮）线上打包 CSS 未做类名混淆、且无 url() 依赖 → 整文件复用是像素级还原的最短路径。
  **为什么**：Vite 默认只压缩不改类名；该项目样式全走 class。
  **怎么应用**：重写组件时 className 必须与分析文档里的原类名一字不差，样式一律不重写。

- **E-003**（第1轮）复刻前先锁依赖版本（React 19.2.5 / three 0.174 / lucide 0.475，均从打包产物里侦察所得），避免行为差异带来的隐性返工。

- **E-004**（第2轮）后台多 agent 并行时可能整批遇到 API 连接中断，属于环境抖动而非任务本身问题。
  **下轮怎么避免**：不要重写任务，直接用工作流的断点续跑(resumeFromRunId)——成功的走缓存零成本，失败的原样重跑。

- **E-005**（第3轮）“能构建”不代表页面可用；一个两行占位组件也会让 Vite 全绿。
  **下轮怎么避免**：除构建外，必须对每个入口做 SSR 冒烟，并遍历所有发布作品和全部节点渲染。

- **E-006**（第3轮）剧情 JSON 是跨创作台与播放器的外部输入，顶层能解析不代表嵌套字段形状安全。
  **下轮怎么避免**：在创作台即时校验和发布 QA 拦截错误类型，同时在播放器加载边界再规范化一次；消费函数仍做防御式判断。

- **E-007**（第3轮）`localStorage` 会因隐私策略、容量或坏 JSON 失败；多个键的操作还可能只成功一半。
  **下轮怎么避免**：读操作降级、写操作显式报错；删除等跨键操作先写主数据，后续失败时补偿回滚，绝不显示假成功。

- **E-008**（第3轮）只检查剧情素材会漏掉落地页和界面音效；只检查“路径非空”还会把不存在的模板 URL 算作完成。
  **下轮怎么避免**：资源测试覆盖源码与剧情引用的全部本地目录；新资产未生成前保持空路径，由摘要和 QA 如实报缺失。

- **E-009**（第3轮）剧情图“可从起点到达”仍可能困在没有结局出口的循环里。
  **下轮怎么避免**：除正向可达检查外，从所有结局沿反向边回溯，要求每个起点可达的非结局节点至少存在一条通往结局的路线。
