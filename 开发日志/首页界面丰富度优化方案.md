# 首页界面丰富度优化方案

> 生成日期：2026-07-17。基于三路分析：原版「衍境心界Beta-妙搭」源码、线上站点 https://wbmnbwl.vercel.app/ （已确认与 参考资料/线上镜像 哈希一致）、当前「幻梦」首页源码。
> 定位：作为候选 Level 执行规格，可并入《女性向AI游戏改造执行PRD》；未获产品确认前不动工。

## 一、差距诊断

核心差距不在信息架构，而在**运动量**与**媒体密度**：

- 原版靠 5 组 keyframes（floatY 立绘漂浮、pulse 呼吸光点、scrollX 双速跑马灯、cue 滚动提示、spin 轨道自转）+ 鼠标视差 + 自动循环视频，页面任何时刻都有 3 处以上元素在持续运动。
- 当前「幻梦」首页 47KB 的 落地页-心界.css 里 **0 个 @keyframes**、无视频、无视差；唯一动效是一次性滚动 reveal，且 Hero 首屏连 reveal 都没挂，加载即全静止。
- 三处最刺眼的"本该动却死着"：「直播倒计时 23:40」写死不走字；关系手账度量条的填充被 `display:none` 关掉；`hx-hero-aurora` 命名像极光实为静态渐变。

| 维度 | 原版 | 当前 |
| --- | --- | --- |
| 首屏冲击力 | 三层立绘 floatY 漂浮 + 五层鼠标视差 + 星点背景 + 徽章 pulse，加载即 5 处运动 | Hero 未挂 reveal，全静止；封面静态 img |
| 持续循环动效 | floatY/pulse/scrollX/cue/spin 五组 keyframes | 零 keyframes |
| 媒体密度 | 1 段循环视频 + 立绘/风格图/照片墙 10 余张轮番出场 | 1 封面 + 2 场景图 + 7 立绘；6 张 360° 全景、6 组情绪差分立绘、20 个视频全部闲置 |
| 悬停反馈 | 系统化：卡片上浮+金边+大阴影、图片 scale 1.05-1.08、跑马灯悬停暂停 | 仅 10 处且多为变色；选择卡/记忆卡/路线卡/结局卡不响应鼠标 |
| 氛围光效 | 星点纹理 + 紫金顶光 + 卡片角光 + pulse 呼吸光贯穿全页 | 有分区主题色 radial 打底但全部静止 |
| 拟真 UI 可信度 | 视频 HUD、自动保存状态字等道具式 UI 靠动效撑住"系统在运转" | 倒计时静止、度量条不动、结局解锁无反馈 |

**改造原则**：不是搬回跑马灯和旧性感立绘，而是用女性向「暮色书页」语法重建持续动效层——慢速呼吸光、Ken Burns 场景缓推、情绪差分立绘、数据驱动的真实倒计时与度量条填充。

**可复用底牌**（已在仓库、零采购）：
- 旧版动效实现完整保留为孤儿代码：源码/样式/落地页.css（cue/floatY/pulse/scrollX/spin 全套 keyframes）、旧 落地页应用.jsx 的鼠标视差引擎（git be8cdda^ 可查）。
- 公共资源/panoramas/ninth-seat/ 6 张 360° 全景（audit-vault/control-room/dressing-room/legal-office/media-gallery/stage-replay）——旗舰作最有价值的未用素材。
- portraits/ninth-seat/ 13 张立绘中 6 名角色有情绪差分（-unguarded/-allied/-warm/-accountable/-candid/-reflective）。

## 二、优化步骤

### 第一步（P0）：纯 CSS「呼吸层」急救包 — 0.5-1 天

**目标**：零新素材，一天内让页面从"全静止"变为任何时刻有 2-3 处克制微动效。性价比最高。

任务：
1. 落地页-心界.css 新增第一批 keyframes（参数参考孤儿文件 落地页.css，换心界色板）：`hx-aurora-breathe`（极光 opacity 0.7↔1 + scale 1↔1.04，14s 循环）、`hx-cue-breathe`（scroll-cue 香槟金竖线高度 24↔38px 呼吸，2.4s）、`hx-dot-pulse`（光点 box-shadow 扩散 ≤9px，2.4s）。
2. 极光挂呼吸；两枚浮签（直播倒计时/人类否决权）加发光圆点脉冲；scroll-cue 恢复竖线呼吸。
3. 心界玩家区块.jsx 给 Hero 的标题/文案/封面/浮签补挂 `.reveal`，首屏获得 0.72s 入场（保持 is-motion-ready 门控，无 JS 时默认可见）。
4. 「直播倒计时 23:40」改为真实走字（useEffect + setInterval，归零循环，保持"剧情装置"语义，关闭 aria-live；reduced-motion 时冻结）。
5. 确认新动画被现有 prefers-reduced-motion 通配规则覆盖；跑 玩家首页自测.mjs。

验收：静置 5 秒可见 ≥3 处持续运动；倒计时走字且无"真实直播"误导；减弱动态下全部停止；自测全绿；Lighthouse 分差 ≤2。
风险：pulse 用低饱和香槟金 #e4c78f、扩散 ≤9px，避免原版"金色泛滥"舞台感；组件卸载须 clearInterval。

### 第二步（P0）：Hero 纵深工程 — 1-1.5 天

**目标**：首屏获得原版立绘拼贴级的纵深感，但用女性向慢速小幅参数。

任务：
1. 从 git 历史移植 mousemove 视差引擎到 落地页应用.jsx（归一化 [-1,1]、`[data-par]` 元素 translate、rAF 节流；≤980px 与 reduced-motion 不挂监听）。
2. Hero 四元素标 data-par：极光 -5、封面卡 4、左浮签 8、右浮签 10（系数取原版一半）。
3. 封面包 overflow:hidden 容器，挂 `hx-kenburns`（scale 1→1.06，26s alternate 循环）——"相册缓慢推近"的电影感而非游戏浮动。
4. 浮签挂 `hx-float-soft`（±6px，9s/11s 错峰）。
5. 封面卡 hover：上浮 4px + 背后描边框旋转角加深 + 投影加深。

验收：视差分层位移无长任务；三个循环周期互不同步；窄屏视差关闭无残留 transform；reduced-motion 全禁用。
风险：只写 transform、视差元素 ≤5 个；Ken Burns 超 1.06 会裁掉封面人物脸部，需目检；漂浮 ±6px/9s 是女性向上限，禁止回到原版 ±14px/6s。

### 第三步（P0）：真实选择区舞台升级 — 1.5-2 天

**目标**：把核心"玩法预览"区从静态截图变成"正在环视现场"的动态舞台，直接展示产品真实 360° 能力（对应播放器闲置自动环视，不算假宣传）。

任务：
1. hx-story-stage 换用 panoramas/ninth-seat/control-room-360.jpg 做 CSS 横移环视（`hx-panorama-drift`：0↔-18% alternate 55s）；首页不引入 three.js，零 WebGL。
2. 舞台角加毛玻璃徽章「360° 全景现场 · 可环视」+ 呼吸光点，点击真实跳 /play。
3. 三条对白 blockquote 逐句浮现（--reveal-delay 0/180/360ms 长错峰）；不做打字机（有假生成嫌疑）。
4. hx-choice-card 补 hover/focus-visible：上浮 3px + 青瓷描边 + 序号变亮；保留"预览不会提交选择"脚注。
5. 窄屏断点降级回静态平面图，避免移动端加载超宽大图；全景图 >400KB 压 2048 宽 webp 派生图。

验收：55s 往返环视无跳帧；对白逐句浮现；选项卡键盘可聚焦、触控 ≥44px；390 宽不加载全景大图；徽章直达真实功能。
风险：全景原图体积必须压缩 + 断点分载，否则 LCP 恶化；环视容器 aria-hidden 装饰语义。

### 第四步（P1）：角色群像情绪差分换装 — 2 天

**目标**：利用 6 组已有情绪差分立绘做 hover/focus 交叉淡入，把"展板"变成"会回应你的人物"，直接呼应女性向关系与情绪主题。

任务：
1. 玩家首页模型.js 为角色补 moodPortrait 字段（映射 portraits/ninth-seat/{slug}-{mood}.png）。
2. 角色卡叠两张 img，:hover/:focus-within 交叉淡入 0.6s；情绪图 lazy + 进视口后 requestIdleCallback 预取。
3. hover 追加立绘 scale 1.04 + 底部情绪标签（「卸下防备」「站在你这边」等）上浮出现。
4. 触屏（hover:none）改点按切换或只保留 scale。
5. 13 张 ~2MB PNG 批量转 1200px webp（保留原件），区块图片负载 ~14MB → ≤3MB。

验收：悬停/键盘聚焦均触发换装并出情绪标签；首屏不加载差分图；触屏无悬挂状态。
风险：webp+懒加载+idle 预取缺一不可；情绪标签文案走"信任/卸防"方向，避免暧昧擦边；差分图需人工目检构图；容器固定 aspect-ratio 防 CLS。

### 第五步（P1）：记忆与结局的「数据在生长」动效 — 1-1.5 天

**目标**：把两处数据驱动却呈现为死物的区块变成"你的选择留下了痕迹"，全部动效由真实存档数据触发，天然规避假宣传。

任务：
1. 恢复关系手账度量条 ::after 填充：`hx-meter-fill`（width 0→var(--meter-value)，1.2s），由存档数据注入 CSS 变量；无存档给 12%/18%/9% 低位"初识"示意 + 脚注「随你的选择变化」；进入视口才播（监听 .in class，不另建观察器）。
2. 结局卡解锁态加一次性香槟金高光斜扫（2.8s 播 1 次）+ Sparkles 图标弹入。
3. 记忆卡补 hover（锁定卡不响应，维持"未解锁"静穆语义差）。
4. 「已解锁 N」数字进视口 600ms 从 0 计到 N（rAF 十行代码，不引库；reduced-motion 直出终值）。
5. 同步更新 玩家首页自测.mjs。

验收：填充值与 localStorage 一致；解锁新结局回首页有一次性斜扫不循环；全部动效 opacity/transform/background-position，CLS 为 0。
风险：无存档示意值必须明显是"起点"（低位+虚线+脚注），不得伪装成已有进度；斜扫金色透明度 ≤0.35，避免"抽卡出金"氪金观感。

### 第六步（P2）：全局氛围收尾与性能预算固化 — 1.5-2 天

任务：
1. 全局两团环境光挂 `hx-ambient-drift`（translate 3-4% + opacity ±0.04，46s/60s 双速）——原版 EvoMap spin"系统在运转"感的女性向替身。
2. 结尾 CTA 背景挂反向 Ken Burns（34s），形成"影院熄灯前最后一幕"。
3. reveal 抽成共享 hook（源码/落地页/使用滚动显现.js），/worlds 与 /creators 接入同款滚动叙事。
4. 移动端 hx-mobile-cta 滚过 Hero 后滑入（0.3s）。
5. 性能预算固化进自测：首页图片总量 ≤4MB、无 WebGL、keyframes 只允许 opacity/transform/background-position（CSS 静态扫描断言）。
6. 320×568/390×844/桌面三档目检 + 键盘走查焦点序。

验收：环境光与极光呼吸周期互质不同步；二级页 reveal 行为一致；CSS 动画属性白名单断言通过；低端设备（CPU 4x 降速）滚动 >30fps，必要时视口外暂停无限动画。

## 三、全程红线

1. **无障碍合同不破**：prefers-reduced-motion 全量降级、is-motion-ready 门控（无 JS 首屏可见）、:focus-visible、44×44 触控、SSR 契约测试全绿。
2. **不做假宣传**：动效不得暗示实时 AI 生成；倒计时保持剧情装置语义；360° 徽章必须直达真实功能；无存档数据不伪装进度。
3. **女性向调性**：低饱和香槟金、慢周期小幅度（呼吸 ≥2.4s、漂浮 ≤±6px、Ken Burns ≤1.06）；不回搬旧男性向素材（landing/char-*.webp、style-*.webp、酒会视频复用需逐张人工把关）。
4. **性能**：所有动画只用 opacity/transform/background-position 走合成层；媒体一律压缩派生 + 懒加载 + 断点分载。

## 四、总量估算

6 步共约 **8-10 个工作日**；P0 三步（约 3-4.5 天）完成后首页动态丰富度即可达到与原版同级的"任何时刻都有元素在运动"，P1/P2 补足媒体密度与全局闭环。每步可独立执行、独立验收、独立提交。
