# 衍境 · 本地复刻

这是「衍境 · 互动影游智能体创作系统」的 Vite + React 本地版，包含产品落地页、互动影游播放器和浏览器本机创作台。剧情与多媒体资源默认从 `公共资源/` 读取，播放器在断网时仍可使用本地内容。

## 本地运行

需要 Node.js 18 或更高版本。

```bash
npm install
npm run dev
```

默认地址为 `http://127.0.0.1:5173`。主要入口：

- `/`：产品落地页
- `/play` 或 `/game`：播放器；可用 `?game=<slug>` 指定作品
- `/play?game=ninth-seat`：《第九席》Level 2 第一章直达入口
- `/creator`：创作台

## 检查与构建

```bash
npm test
npm run build
npm run preview
```

`npm test` 会运行剧情引擎、创作台、《第九席》内容路线、资源引用和页面壳层回归；生产文件输出到 `dist/`。

## 数据与浏览器存储

- 静态作品：`公共资源/games/<slug>/story.json`
- 首页精选：`公共资源/showcase.json`
- 全景、视频、语音、音乐和界面素材：`公共资源/` 下对应目录
- 创作台本机项目、精选设置与模型密钥：仅保存在当前站点的浏览器 `localStorage`
- 玩家存档：按作品分别保存在当前站点的浏览器 `localStorage`

清理站点数据会同时删除本机创作项目、设置和玩家存档。不要把真实 API Key 写进源码或提交到仓库。

## 部署注意

三个页面由前端按 URL 路径分发。部署到静态托管平台时，需要把 `/play`、`/game`、`/creator` 及其尾斜杠形式回写到 `index.html`，否则直接刷新子路径可能由托管平台返回 404。

创作台当前已实现本机项目管理、节点编辑、校验、精选管理和发布到本机播放器。依赖 DeepSeek、图片、语音或音乐服务的生成能力若仍显示“尚未接入”，代表该外部调用尚未启用，不会在后台偷偷发送密钥或产生费用。
